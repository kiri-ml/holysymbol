# Product

## Register

product

## Users

MapleLegends leech sellers who need to track live leech runs and calculate buyer charges quickly during or after a run. They are likely juggling in-game activity, buyer details, timers, level/EXP snapshots, and quote expectations, so the interface needs to stay fast, legible, and hard to misread.

## Product Purpose

Holy Symbol is a MapleLegends leech calculator for quoting runs, tracking buyers, recording start and current EXP snapshots, timing hourly sessions, calculating mesos due, and exporting run records. Success means sellers can produce accurate charges with minimal friction while a run is active.

## Brand Personality

Precise, fast, trustworthy. The product should feel like a reliable run ledger: direct, task-focused, and confident without becoming sterile or intimidating.

## Anti-references

Avoid playful game-fan decoration, marketing-site hero treatment, and dense spreadsheet UI that slows down live run tracking. The app should not bury common actions behind decorative layouts or make billing feel approximate.

## Design Principles

- Keep calculation confidence visible: important inputs, billing mode, timer state, EXP gained, and mesos due should be easy to verify at a glance.
- Optimize for live-run speed: common actions should be reachable without scanning decorative or unrelated content.
- Make state changes explicit: fetching, manual entry, timer transitions, locked snapshots, and destructive actions should have clear feedback.
- Preserve seller focus: density is useful when it supports repeated tracking, but the hierarchy should prevent spreadsheet-like fatigue.
- Favor familiar controls: standard inputs, buttons, tables/lists, and inline affordances should carry the workflow.

## Accessibility & Inclusion

No special requirements beyond solid product defaults. Target WCAG AA contrast for text and controls, preserve keyboard accessibility for interactive workflows, and respect reduced-motion preferences.
